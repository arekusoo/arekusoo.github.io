Super Monitoria

Este projeto se define por um sistema de monitoria para alunos, monitores, coordenadores e professores tem como objetivo oferecer uma plataforma online para facilitar a comunicação e o acompanhamento das atividades de monitoria em uma instituição de ensino. Abaixo estarão detalhados as etapas realizadas no desenvolvimento do projeto. Os objetivos do projeto foram definidos através de um briefing direto com o cliente, que por sua vez, solicitou que o projeto devesse ser produzido como uma plataforma digital. Participei deste projeto como Designer de UI e UX, além de definir conceitos de branding.

Objetivo 1: Conhecer os papéis envolvidos e as necessidades de cada um.

Objetivo 2: Desenvolver um sistema digital que auxilie os papéis de acordo com suas necessidades.

1. Entender

Na primeira etapa foi realizado o entendimento do projeto e dos usuários, com o auxilio de uma Matriz CSD foi possível de guiar os participantes e alinhar as expectativas do produto final. Além disso, foi possível mapear as principais dúvidas em relação ao projeto e aos usuários.

De acordo com as dúvidas e as suposições levantadas na matriz sobre o projeto as seguintes questões se alinharam aos objetivos.
- Qual a necessidade de uma plataforma digital?
- Como um processo de monitoria é feito atualmente e qual o impacto de uma plataforma digital?
- Qual o público-alvo do sistema? Qual a necessidade dos envolvidos do sistema?

Para responder as perguntas acima, foi aplicado um formulário para:
Os alunos, onde o objetivo era entender suas dificuldades no formato atual de agendar uma monitoria caso precisasse.
Os monitores, onde teve um objetivo similar ao do usuário aluno.
Os professores, para saber como eles gerenciam as monitorias realizadas.

Além do formulário, foi realizado uma pesquisa de similares apoiado em uma análise SWOT dos produtos a fim de entender como as demais instituições tratam desse problema e como é feito o processo de agendamento. Dentre os produtos pesquisados estão: Toolzz Mentor, Mentor Cruise e Trello.
O sistema tem como cliente principal o coordenador, então uma entrevista foi realizada com o intuito de mapear suas necessidades para o sistema.

Análise SWOT

Análise da Toolzz Mentor, uma plataforma online para agendamento de monitoria:

Forças:

Personalização: oferece recursos avançados de personalização que permitem aos usuários configurar o agendamento de monitoria de acordo com suas necessidades.
Recursos específicos para agendamento de monitoria: tem recursos específicos para gerenciar agendamentos de monitoria, como atribuição de tarefas e acompanhamento do progresso.
Interface intuitiva: aparenta ser fácil de usar que permite aos usuários configurar e gerenciar agendamentos de monitoria sem complicações.

Fraquezas:

Dependência da Internet: como uma plataforma online, a plataforma depende da Internet para funcionar adequadamente, o que pode ser um problema se a conexão estiver lenta ou interrompida.
Integrações limitadas: a plataforma pode não ter tantas integrações com outros aplicativos populares como outras ferramentas de gerenciamento de tempo.

Oportunidades:

Crescimento do mercado de educação online: com o aumento da demanda por soluções de educação online, há uma oportunidade de crescimento.
Integrações com outras ferramentas: pode explorar parcerias com outras ferramentas de gerenciamento de tempo ou aplicativos de produtividade para expandir seu alcance e funcionalidade.
Recursos adicionais: pode continuar a adicionar recursos adicionais, como análise de dados e relatórios para fornecer mais insights sobre o progresso da monitoria.

Ameaças:

Tecnologia em mudança: a tecnologia está sempre mudando, então pode haver novas ferramentas ou soluções de gerenciamento de tempo que podem ameaçar a posição plataforma como uma ferramenta popular para agendamento de monitoria.

Após os dados analisados, foi possível mapear requisitos de usabilidade para o sistema, onde esses foram pontos de partida para a interface a ser desenvolvida.

Requisitos de Usabilidade

Facilidade de uso:o sistema deve ser fácil de usar e navegar, com um layout claro e intuitivo que permita aos usuários realizar suas ações com rapidez e eficiência.

Feedback constante: o sistema deve fornecer feedback aos usuários sobre suas ações através de respostas pela plataforma.

Compatibilidade com dispositivos móveis: o sistema deve ser compatível com dispositivos móveis para permitir que os usuários acessem a plataforma.

Notificações automáticas: o sistema deve enviar notificações automáticas por e-mail para avisar os usuários quaisquer informações relevantes.

Facilitar o contato entre os usuários: o sistema deve fornecer uma maneira fácil para que os usuários tenham acesso aos dados de contato após o agendamento da monitora.
